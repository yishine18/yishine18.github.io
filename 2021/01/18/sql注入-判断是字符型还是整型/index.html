sql注入-判断是字符型还是整型
ハッカー
2021-01-18 00:40:24
  • sql注入
  • sql_injection

?id=1 and 1=2

来自百度：

总结这个问题如下：
SELECT uname,email FROM userinfo WHERE id = ‘1 and 1=3’ ;
这句SQL是可以查询到结果的,当然前提是有id=1的数据。
其实，1后面的字符可以是任意的，不会影响查询，它甚至可以是：
SELECT uname,email FROM userinfo WHERE id = ‘1dsasasdasasd’ ;
那么为什么会这样呢？
经过测试，我发现这是因为id的类型为数值型,例如int，而查询的时候传入的是字符串，而MYSQL是十分智能的，它会进行类型转换，但是这个转换过程是很随意的，就像varchar 的’2asdasd’会被转成int的 2。
当然，如果id本身就是varchar类型的，那么数据库查询的时候就会进行精准的匹配，这样子’2adsdasd’等类似的数据由于数据库里没有匹配项,查询结果为空。

卧槽是正确的
